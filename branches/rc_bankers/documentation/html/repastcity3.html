RepastCity - A demo virtual city (version 3)

Introduction

RepastCity is a small Repast Simphony program which demonstrates how to create a virtual city and move some agents around a road network. Although the code to move around a road network is quite complicated, most of the details can be ignored and developers can concentrate on implementing agent behaviours. This document (as well as the extending and model_structure documents) briefly describe how the program works and explains how it can be adapted for different applications (my own research interest is crime simulation). More (or updated) information can be found on the project's website: http://code.google.com/p/repastcity/

People who are new to Repast Simphony definitely need to work through some tutorials first. The ones on the Repast Simphony website are excellent (in particular I would recommend the Java tutorial). Repast Simphony has quite a steep learning curve but is well worth learning if you need to build agent-based models, it's an excellent tool.

Getting the Program and Running It

The program is available as a single compressed file which contains the entire Eclipse project. A link is available on the project website. Download the file and extract it somewhere (the location is not important).

To run the program you need to download Repast Simphony version 2.0 (it comes bundled with the Eclipse development environment) and then follow these instructions:

  1. Start Repast (Eclipse)
  2. Go to File -> Import
  3. Under 'General' select 'Existing Projects Into Workspace'.
  4. Go to 'Select root directory' click on browse and select the extracted project.
  5. Click on 'Finish'. If you left the 'copy projects into workspace' box ticked you can now delete the zip file and the extracted directory because all the code will be copied into your Eclipse workspace.
  6. Finally click on the 'Run' button (green) at the top of Eclipse and select 'Run RepastCity3 Model'. (If there is not option to run the model try closing the project and then re-opening it).
